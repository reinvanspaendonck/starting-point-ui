## Components

- [Input](/docs/components/input) — Single-line text entry for text, email, password, and other types
- [Input Group](/docs/components/input-group) — Decorate inputs with icons, text, or other addons
- [Textarea](/docs/components/textarea) — Multi-line text input
- [Select](/docs/components/select) — Native dropdown for choosing from a list of options
- [Combobox](/docs/components/combobox) — Searchable, keyboard-navigable dropdown with single and multi-select
- [Checkbox](/docs/components/checkbox) — Toggle options on and off, with card and list select variants
- [Radio Group](/docs/components/radio-group) — Select a single option from a group, with card and list select variants
- [Switch](/docs/components/switch) — Toggle settings on and off
- [Label](/docs/components/label) — Accessible labels for form fields
- [Button](/docs/components/button) — Actions and form submissions
- [Button Group](/docs/components/button-group) — Group related buttons together

## Accessibility

All form components support standard accessibility patterns:

- Use [Label](/docs/components/label) with matching `for` and `id` attributes to connect labels to inputs
- Use `aria-invalid="true"` for validation error states
- Use `aria-describedby` to connect error messages to their fields
- Use `disabled` to prevent interaction on any form element
- Use `peer` on disabled toggles so labels dim automatically via `peer-disabled`
